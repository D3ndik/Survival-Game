# Mobile Roguelike Bullet-Heaven / Auto-Battler Prototype (Unity 2D)

[![Unity Version](https://img.shields.io/badge/Unity-2022.3%2B-blue?logo=unity)](https://unity.com/)
[![Platform](https://img.shields.io/badge/Platform-Android%20%7C%20PC-green?logo=android)](https://developer.android.com/)
[![Language](https://img.shields.io/badge/Language-C%23-purple?logo=csharp)](https://learn.microsoft.com/en-us/dotnet/csharp/)

Прототип мобильной кроссплатформенной 2D-игры в жанре **Roguelike Bullet-Heaven / Автобаттлер** (в стиле *Vampire Survivors*), разработанный в рамках практической подготовки. Основной фокус проекта направлен на построение гибкой архитектуры управления данными, оптимизацию оперативной памяти под мобильные платформы и реализацию бесшовного кроссплатформенного ввода.

---

## 🎮 Игровой процесс (Core Loop)
- **Непрерывное маневрирование:** Игрок управляет перемещением персонажа на замкнутой арене, уклоняясь от наступающих волн противников.
- **Автоматический бой:** Атака врагов и стрельба снарядами осуществляются полностью в автоматическом режиме по заданным алгоритмам поиска ближайших целей.
- **Прогрессия:** Уничтожение противников генерирует кристаллы опыта. Сбор опыта заполняет шкалу прогресса, и при повышении уровня (`Level Up`) игроку предоставляются улучшения и дополнительное автоматическое оружие.

## 🛠️ Технические особенности и оптимизация

В ходе разработки были решены ключевые инженерные задачи по оптимизации производительности программного кода:

1. **Паттерн Object Pool (Пул объектов):**
   Полный отказ от ресурсоемких стандартных вызовов `Instantiate()` и `Destroy()` во время активного геймплея. Спавн противников, снарядов оружия и частиц опыта реализован через переиспользование объектов в пуле. Это исключило пиковые нагрузки на сборщик мусора (**Garbage Collector**) и обеспечило стабильный FPS при наличии **100+ активных сущностей** на сцене.
   
2. **Архитектура Data-Driven Design:**
   Характеристики игровых сущностей, параметры баланса оружия и свойства апгрейдов изолированы от кода и не имеют жесткого кодирования («хардкода»), что позволяет гибко масштабировать проект.
   
3. **Оптимизация физики 2D:**
   Компоненты `BoxCollider2D` и `CircleCollider2D` снарядов и врагов переведены в режим триггеров (`Is Trigger`). Взаимодействие рассчитывается без применения тяжелых законов классической твердотельной динамики, что позволило реализовать линейное прохождение снарядов сквозь массив объектов с параллельной дискретной регистрацией урона.

4. **Рефакторинг кода поиска объектов:**
   Устаревшие и ресурсоемкие методы поиска объектов на сцене вида `Object.FindObjectOfType()` были заменены на современные оптимизированные вызовы `Object.FindAnyObjectByType()`.

## 📱 Мобильная адаптация и UI

Проект полностью оптимизирован под мобильные устройства под управлением ОС Android:
- **Кроссплатформенный ввод (Active Input Handling):** В настройках проекта активирован режим `Both`, позволяющий бесконфликтно обрабатывать ввод как с классической ПК-клавиатуры (**WASD**), так и с экранного сенсорного интерфейса.
- **Виртуальный джойстик:** Интегрирован компонент `Fixed Joystick` (фиксированный джойстик) для точного позиционирования пальца на сенсорном экране.
- **Фиксация ориентации экрана:** В параметрах `Resolution and Presentation` заблокирован вертикальный (портретный) режим. Включен автоповорот исключительно для альбомных ориентаций (`Landscape Left`, `Landscape Right`).
- **Адаптивный UI Canvas:** Архитектура графического интерфейса пользователя спроектирована с использованием адаптивных якорей (`Anchors`) и `Rect Transform`. Корректно отображаются:
  - **Главное меню (Main Menu)** с логикой переходов между сценами.
  - **Игровой HUD:** Динамические слайдеры здоровья (`HP`), шкала опыта (`XP Bar`) и текстовый счетчик убийств (`Kills UI`).
  - **Экран паузы (Pause Menu):** Реализована остановка игрового времени через манипуляции с масштабом времени (`Time.timeScale = 0f`).
  - **Экран окончания игры (Game Over):** С выводом итоговой статистики игровой сессии и кнопкой перезапуска.
